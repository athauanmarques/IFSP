Lição 17: As coisas invisíveis: Scripts, Metadados, viewports e Comentários

Nesta lição, você vai entender mais sobre como controlar coisas invisíveis ... que são realmente importantes.

Scripts

Tags de script são uma maneira fácil de conectar JavaScript ou quaisquer outros tipos de linguagem de programação em suas páginas HTML5.

Eles agora são também muito fácil de usar. Em algum lugar do corpo de sua página, adicionar o elemento de script da seguinte forma:

Exemplo de roteiro

	
	<Script>
	  document.write ("Olá a partir HTML.net!")
	</ Script>

Salve o arquivo e usuário o seu navegador para experimentá-lo. Não é chique, mas qualquer tipo de script pode ser colocado aqui, a partir de algo que calcula os valores para algo que executar controles vídeos.

Se você queria voltar a usar um script uma e outra vez em várias páginas, você iria colocar esse script em um arquivo .js - apenas salvar um arquivo com os js de extensão em vez de .htm - e se referem a ele a partir do elemento de script sempre que você precisar.

Exemplo de script externo

	
	<Script src = "MyScript.js" />

HTML5 pressupõe que qualquer roteiro é por padrão JavaScript, por isso, se você estiver usando outro idioma, você teria que definir o tipo de atributo também.

Você também pode colocar os scripts que controlam a página inteira no <head> ou <footer> elementos vez. Algumas das melhores práticas defensores dizem que colocar os scripts de trabalho intensivo no elemento de rodapé, logo antes do </ body> tag ajuda mais rápido que o navegador do carregamento da página ler e mostrar o HTML simples e CSS antes de começar sobre os cálculos mais complicados no script.

Metadados

Os documentos contêm mais do que apenas o código e o conteúdo - que também deve incluir metadados, que é a informação sobre a página. Esta informação não está visível para os leitores, mas se acostumar pelo navegador e pelos motores de busca.

Colocamos metadados no <head> elemento, onde ele pode ser lido rapidamente pelos motores de busca.

O <meta> tag tem uma série de atributos e é considerada uma boa prática para adicionar as quatro linhas de seguir <meta> tags para cada página, mas alterando o valor de conteúdo em cada caso, para coincidir com o tipo de conteúdo que você tem em que página e quem a escreveu.

Exemplo de metadados para dar a cada página

	
	<Head> 
	  <name = "description" content = "meta cursos livres HTML5"> 
	  <Meta name = "keywords" content = "HTML, CSS, JavaScript"> 
	  <Name = conteúdo "autor" meta = "HTML.net"> 
	  <Meta charset = "UTF-8"> 
	</ Head>

Quanto mais exatas palavras-chave e descrição, o mais provável é que sua página vai voltar no topo da lista de um motor de busca de resultados quando alguém procura por esses termos particulares.

O charset atributo é novo para HTML5 e permite que você especifique o conjunto de caracteres que você está usando. Ele substitui esta linha em HTML4: <meta http-equiv = conteúdo "Content-Type" = "text / html; charset = utf-8"> . Para Inglês, o valor é sempre UTF-8. Se o seu conteúdo é cirílico, grego, francês ou qualquer outro conjunto de caracteres, você precisará configurá-lo de acordo (alfabeto latino, por exemplo, tem a certificação ISO-8859). Você pode verificar esta tabela para o código de direito se você estiver usando um outro conjunto de caracteres.

Viewports

Há mais uma coisa a acrescentar em seus metadados no <head> elemento e isso é uma linha que garante que o seu conteúdo é pouco visível, não importa que tipo de dispositivo que alguém está usando para navegar no seu site: smartphone, tablet, laptop, ou qualquer outra coisa então eles podem usar.

	
	<Head> 
	  <name = "description" content = "meta cursos livres HTML5"> 
	  <Meta name = "keywords" content = "HTML, CSS, JavaScript"> 
	  <Name = conteúdo "autor" meta = "HTML.net"> <meta charset = "UTF-8"> 
	  <Meta name = "viewport" content = "width =-largura do dispositivo, escala inicial = 1,0"> 
	</ Head>

O valor do conteúdo define a largura da página para a largura padrão que o navegador do dispositivo suporta e também define o nível de zoom a partir de um valor que vai fazer a maior parte do conteúdo visível na maioria dos aparelhos, vai mudar e atualização se girar o dispositivo, embora também vai ser capaz de aumentar o zoom. Há muitas outras propriedades possível aqui, como a criação de uma largura máxima em pixels, mas os valores que temos dado são uma boa prática reconhecida (tão bom como podemos obter) neste momento.

Comentários

Às vezes pode ser muito útil para deixar-se uma nota no código. Talvez seja porque você está organizando as coisas ou talvez seja porque você precisa se lembrar por que você fez alguma coisa em particular ou que fez algo - quem sabe. É realmente fácil de adicionar em um comentário que não seja analisado pelo navegador - o navegador ignora direito sobre ele.

Como tudo mais, um comentário HTML deve estar dentro de seu próprio conjunto de suportes angulares, mas o suporte aberto é imediatamente seguido pelo! símbolo e dois traços. Para fechar um comentário, adicionar mais dois traços e um outro ângulo de montagem. Você pode ter a quantidade de texto entre os símbolos que você precisa, incluindo outras tags.

Exemplo de comentários HTML

	
	<! - Me lembrando de verificar este código para validade HTML5 -> 

	<! - Atualizado por AA em 1 de Julho -> 

	<! - Este é um pedaço de código que eu estou indo para solucionar <title> Notas </ title> ->

Você já sabe como controlar todas essas coisas invisíveis que você pode nunca ter conhecido existiu, mas que são muito importantes e úteis.
