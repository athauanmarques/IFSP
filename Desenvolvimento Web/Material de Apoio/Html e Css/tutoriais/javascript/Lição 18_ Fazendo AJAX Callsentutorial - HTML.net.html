Lição 18: Fazer chamadas AJAX

Por Maria Antonietta Perna

Uma característica central de aplicativos web do século 21 é a sua fantástica capacidade de resposta. Um pioneiro no negócio de fazer a web mais e mais semelhante a um aplicativo de desktop tem sido Google : assim que você começar a digitar um caractere em uma caixa de Google, todos os tipos de sugestões e resultados de pesquisa continuam surgindo para acelerar a sua pesquisa; e, novamente, basta digitar algo usando o Google Maps, e você está imediatamente catapultou para quase qualquer rua do planeta.

Por trás dessa incrível façanha de inteligência humana é a união de algumas tecnologias conhecidas coletivamente como AJAX .

Nesta lição você vai aprender:

  • o que AJAX significa e o que tem tudo a ver ;
  • como fazer uma solicitação AJAX ;
  • como lidar com uma resposta AJAX .

Eu recomendo que você tenha acesso a um servidor web para realizar o código de exemplo ilustrado nesta lição. Na verdade, AJAX é tudo sobre comunicação cliente-servidor. Em vista disso, você pode fazer upload de seus arquivos para um servidor hospedado ou em um servidor local. O tutorial PHP neste site oferece um excelente guia sobre este tema.

O que é AJAX tudo isso?

A sigla significa AJAX Asynchronous JavaScript And XML .

Na realidade, é uma combinação de padrões da Internet composta por:

  • baseada em padrões de apresentação usando HTML e CSS ;
  • exposição dinâmica usando o DOM ;
  • intercâmbio de dados e manipulação usando XML ;
  • assíncrona recuperação de dados usando o objeto XMLHttpRequest ;
  • JavaScript mágica para orquestrar todo o processo.

Em aplicações web não-AJAX, a interação entre os servidores e clientes pode ser um negócio tedioso:

  1. uma ação do usuário a partir do cliente envia uma solicitação para o servidor web via HyperText Transfer Protocol (HTTP);
  2. o servidor web recebe o pedido e processa-lo invocando scripts do lado do servidor, os dados do banco de dados, etc., e envia uma resposta de volta para o cliente via HTTP;
  3. o navegador do cliente recebe a resposta e carrega toda a página atualizada.

Ter que ir de navegador para o servidor e voltar cada vez que um usuário solicita algum pedaço de dados do servidor, além de passar por toda uma atualização de página em cada atualização, pode ser muito estressante em servidores e em usuários da mesma forma.

AJAX ajuda em pelo menos dois aspectos: atualizações da página fragmentada e comunicação assíncrona entre servidor e cliente.

O que isto significa, em poucas palavras, é que cada vez que o usuário interage com a página da Web, apenas os bits que precisam de atualização são atualizados, e não toda a página. Além disso, o facto de as operações são realizadas de forma assíncrona AJAX, significa que, durante o tempo que leva para a resposta do servidor, a página não é fechado e o utilizador pode ainda interagir com o sítio.

Como posso fazer um pedido de AJAX?

Um pedido AJAX é feita usando o objeto XMLHttpRequest e seu open () e enviar () métodos . Isto é suportado por todos os principais navegadores. No entanto, os navegadores mais antigos, varsions a saber mais antigas do Microsoft Internet Explorer (vesions 5 e 6), apoiar um ActiveXObject . Este pequeno obstáculo é facilmente superada por meio de testes para suporte de recursos no script.

O (, url, retrievalMethod bool) aberto método tem três argumentos:

  1. retrievalMethod : este pode ser um GET (usada para obter os dados a partir do servidor), ou um POST (utilizado para enviar dados para o servidor);
  2. url : esta é a localização onde os dados são disponibilizados. Pode ser um arquivo de texto, um documento XML, ou um script do lado do servidor que processa os dados provenientes de um banco de dados;
  3. bool : este é um valor verdadeiro / falso. Se é falso o pedido for feito de forma síncrona, se é verdade o pedido for feito de forma assíncrona, que é o que normalmente quer.

O objeto XMLHttpRequest tem uma onreadystatechange propriedade que lida com a resposta do servidor. Isso procede ao longo dos 5 estágios seguintes:

  • 0) o pedido não foi inicializado porque open () não foi chamado ;
  • 1) o pedido é especificado , mas send () não foi chamado ainda ;
  • 2) o pedido está sendo enviado , porque agora enviar () foi chamado ;
  • 3) a resposta está sendo recebido, mas ainda não está concluída ;
  • 4) a resposta é completa e os dados estão disponíveis para manipulação e exibição .

Após a conclusão (estágio 4), o objeto XMLHttpRequest propriedade status é atribuído um código de status HTTP que descreve o resultado do pedido da seguinte forma:

  • 200 : sucesso!
  • 401 : não autorizado - a autenticação é necessária e não foi fornecido;
  • 403 : proibido - o servidor se recusa a responder;
  • 404 : não encontrado - o recurso solicitado não pode ser encontrado.

Este é um trecho de código simples que traduz o que foi dito em prática:

		
		// Variável global para armazenar o objeto XMLHttpRequest
		
		var myRequest;
		
		// Package o pedido em sua própria função
		
		funcionar getData ()
		
		{
		
		// Matéria técnica de testes para garantir que o navegador
		
		// Suporta o objeto XMLHttpRequest
		
		if (window.XMLHttpRequest)
		
		{
		
		// Cria uma nova instância do objeto
		
		myRequest = new XMLHttpRequest ();
		
		}
		
		// O resto - o objeto XMLHttpRequest não é suportado:
		
		// Criar uma instância de ActiveXObject
		
		mais
		
		{
		
		myRequest = new ActiveXObject ("Microsoft.XMLHTTP");
		
		}
		
		(, RetrievalMethod "myDataFile.txt" bool) método // Use a céu aberto
		
		myRequest.open ("GET", url, true);
		
		// Use enviar () com um argumento nulo - não temos nada a enviar:
		
		myRequest.send (null);
		
		// Anexar uma função para lidar com onreadystatechange,
		
		// Isto é, a resposta do servidor:
		
		myRequest.onreadystatechange = getData;
		
		}

Como posso lidar com a resposta AJAX?

Se o navegador do cliente solicita dados de texto, a resposta do servidor é automaticamente armazenada na propriedade responseText .

Se o navegador do cliente solicita dados em formato XML, a resposta do servidor é armazenado na propriedade responseXML .

Aqui está um trecho de código que ilustra isso na prática.

		
		// Package o código de resposta a movimentação em uma função
		
		funcionar getData ()
		
		{
		
		// Certifique-se a resposta está em fase de conclusão (4):
		
		se (myRequest.readyState === 4)
		
		{
		
		// Se for, certifique-se o código de status é de 200 (sucesso):
		
		se (myRequest.status === 200)
		
		{
		
		// Se está tudo bem, lidar com a resposta:
		
		var text = myRequest.responseText;
		
		alert (texto);
		
		}
		
		}
		
		}

		// Prepara a variável global para a solicitação
		
		var myRequest;
		
		// Escreve a função getText (url)
		
		função getText (url)
		
		{
		
		// Verificar o suporte para o objeto XMLHttpRequest
		
		if (window.XMLHttpRequest)
		
		{
		
		myRequest = new XMLHttpRequest ();
		
		}
		
		// Mais, criar um ActiveXObject para IE6
		
		mais
		
		{
		
		myRequest = new ActiveXObject ("Microsoft.XMLHTTP");
		
		}
		
		// Chame o método open () para fazer o pedido
		
		myRequest.open ("GET", url, true);
		
		// Envia o pedido
		
		myRequest.send (null);
		
		// Atribuir a função getData () para o
		
		// Propriedade onreadystatechange para lidar com a resposta do servidor
		
		myRequest.onreadystatechange = getData;
		
		}
		
		
		
		/ ********************************************** /
		
		
		
		// Essa função manipula a resposta do servidor
		
		funcionar getData ()
		
		{
		
		// Obter uma referência para o elemento de cabeçalho onde
		
		// O resultado retornado será exibido
		
		var MyHeader = document.getElementById ("MyHeader");
		
		// Verificar a resposta está completa
		
		se (myRequest.readyState === 4)
		
		{
		
		// Verificar o código de estado da resposta é bem sucedido
		
		se (myRequest.status === 200)
		
		{
		
		// Conservar a resposta
		
		var text = myRequest.responseText;
		
		// Assing o texto retornado para o nodeValue

		}
		
		}
		
		}

Resumo

Isso é tudo para esta lição. Você trabalhou muito duro, mas agora você pode escrever código JavaScript que recupera dados do servidor, sem uma atualização de página inteira usando técnicas AJAX.

Você não está limitado a arquivos de formato de texto ou arquivos XML, no entanto. Na verdade, os bancos de dados são os instrumentos mais utilizados para armazenar permanentemente os dados em um servidor. Técnicas AJAX são tão facilmente aplicada para lidar com banco de dados de informações recuperadas. No entanto, você vai precisar de PHP ou uma linguagem de programação do lado do servidor para recuperar dados de ou postar dados para um banco de dados.

Em todo caso, vamos voltar a este tema em nossa última lição e ver como jQuery simplifica bastante todo o processo Ajaxification .

Mas, primeiro, precisamos ter alguma familiaridade com a biblioteca mais popular do mundo de programação JavaScript hoje, jQuery , que é o tema da nossa próxima lição.
