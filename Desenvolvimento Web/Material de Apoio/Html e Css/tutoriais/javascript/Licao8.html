Funções: o que são e para o que servem

Nas lições anteriores, você usou funções JavaScript para executar algumas ações de forma rápida e facilmente, como a geração de números aleatórios, escrevendo algo em uma página web, etc Na verdade, JavaScript tem um grande built-in funções como write(), alert(), getElementById (), random(), floor() , e vários outros. Sempre que você encontrar um par de parênteses há provavelmente uma função em ação.

JavaScript também lhe dá a opção de criar suas próprias funções.

Uma função é uma forma de embalar seus comandos de JavaScript para que você possa facilmente reutilizá-los cada vez que você precisar do mesmo pedaço de funcionalidade implementada em seu site .

Isto sugere que há duas fases distintas para uma função:

  1. A fase em que a função é declarada (criada) ;

  2. A fase em que a função é chamada (usada) .

Vamos passar por cada fase de separadamente.

Declarar uma função

A estrutura básica de uma função JavaScript parece com isso: 

	
	    // Escolher nome para a função    
	    // Não esqueça os parênteses!  
	    function   functionName ()  
	    {    
	    // Código que você deseja executar vai aqui    
	    }

Chamada de uma função

Uma vez que você colocou um bloco de código em uma função, você pode usá-lo em qualquer lugar que você precisa que a funcionalidade em seu site. O que você precisa fazer é apenas chamar a função .

Você pode fazer isso digitando o nome da função seguido por colchetes e a função irá fazer o seu trabalho - tal como fez com alert(). Veja como é feito: 

	
	  // o nome da função (sem a palavra-chave function)  
	  //  Não esqueça os parênteses!  
	  functionName();

Isso é tudo que você precisa: o código que você colocou em sua função é executado neste momento.

Vamos ver como isso funciona na prática. Abra o seu editor de texto: é hora de se programar!

Perguntas, perguntas, perguntas

Eu sei, há algumas coisas novas nos exemplos de código acima. Você pode estar se perguntando: 1) o que é isso new Date () negócio? E 2) O que toDateString () fazer para a data? Vamos resolver cada pergunta por sua vez.

  1. O que é new Date() ?

    A palavra-chave Date cria uma nova instância de um objeto JavaScript , neste caso um objeto Data com base no relógio do computador do usuário. Objetos são o tema da próxima lição, e o objeto Date é o tema da lição 11.

    A única coisa importante que você precisa saber agora é que uma vez que você tenha criado uma instância do objeto Date, você tem todos os tipos de funções úteis (chamados métodos do objeto ) ao seu alcance para manipular data e hora.

  2. Por que eu uso toDateString () com a data?

    Na sequência do ponto anterior, toDateString () é apenas um dos vários métodos que você pode usar com o objeto Data. O interpretador JavaScript lê como segue:

    "Ei navegador, tomar a data em que estive ligado ao e torná-lo um pouco mais humano-amigável!"

    Isto é o que eu quero dizer. Digite o seguinte dentro encerrando marcas <script> de uma página HTML: var myDate = new Date (); document.write (myDate);

    Salve a página e executá-lo no navegador: sua data deve ser exibido em um formato como a abaixo (obviamente eu espero que a data para ser diferente):

    Dom 06 de novembro de 2011 14:45:30 GMT 0000 (GMT Standard Time)

    Feio! Felizmente, o objeto Date tem uma solução para isto. Reescreva o trecho de código anterior da seguinte forma: var myDate = new Date (); document.write (myDate.toDateString ());

    E aqui está como a data acima parece depois do toDateString () :

    Dom 06 de novembro de 2011 .

    Muito melhor, menos assustador, e muito mais legível para nós seres humanos.

Alimentar as informações para e recuperar informações de uma função

Funções para manipular dados: a função alert() exibe mensagens para mostrar como seus dados, a função write() escreve seus dados na página da web, etc

Você também pode passar dados para uma função . A maneira de fazer isso é através de argumentos (ou parâmetros) . Um argumento é colocado entre parênteses da função quando a função é declarada . Você pode colocar um ou mais argumentos, separados por vírgulas (,) dentro de uma função.

Aqui está a estrutura básica: 

	
			    function    functionName (arg1, arg2, arg3)    
			    {    
			    	// Corpo da função vai aqui    
			    }

O argumento da função é como um espaço reservado para o valor que é fornecido quando a função é chamada. Isso vai parecer mais claro no exemplo abaixo.

Finalmente, uma coisa útil que as funções podem fazer com os dados, uma vez que foi manipulado, é devolvê-lo.

Por exemplo, considere a função floor(número) que você já conhece da lição anterior. Esta funçãorecebe como argumento um número e retorna um número inteiro.

Se o seu script faz algo com o valor retornado em seguida, ele precisa atribuir a função a uma variável ao chamar a função.

Vamos colocar esse conhecimento em prática imediatamente. Crie uma função e depois faça uma chamada a função para usar seu valor de retorno em seu documento.

Declarar uma função com um parâmetro

Prepare uma página HTML simples e digite o seguinte código JavaScript dentro das tags <script> ... </ Script>

	    function SomaNumeros(num1, num2)    
	  
	    {    
	  
	         // Esta é uma função simples:    
	  
	         // Apenas retornar a soma dos dois argumentos de número    
	  
	       return num1 + num2;    
	  
	        // Sua função termina aqui    
	  
	    }    
	    
	  </ Script>    
		    
	  </ Body>      
	  </ Html>

Chamar sua função e usar o valor de retorno

Continue a partir do código anterior. Fora de sua função, apenas após a chave de fechamento} , digite o seguinte: 

	
	    // Chamar a sua função   
	  
	    // Criar uma var e atribuir o valor retornado pela sua função a ela    
	  
	    // Além disso, passar dois números como argumentos    
	  
	    // Digitando dois números separados por vírgula entre colchetes    
	  
	    var total = SomaNumeros (3, 5);

Salve seu trabalho e execute a página no navegador. Se tudo correr bem, a soma dos números que você inseriu como argumentos para SomaNumeros() função deve ser exibido na página web.

Argumentos de funções não são limitados a um tipo de dados específicos. Você também pode usar strings e booleanos, e até mesmo outras funções.

Variáveis ​​dentro e fora de escopo funções:

Uma variável pode ter âmbito local ou global, dependendo de ter sido declarada dentro ou fora de um bloco de função. . Mas, o que isso significa exatamente?

Simplificando, uma variável com escopo local significa que é visível, ou acessível, apenas dentro da função em que vive . Nenhuma outra parte do seu código pode ver uma variável local.

Por outro lado, uma variável com escopo global quer dizer que é acessível, isto é, ela pode ser utilizada, em qualquer lugar no script. . Vamos ver isso na prática. Obtenha seu editor de texto pronto e digite o seguinte trecho entre a abertura e fechamento <script> em uma página HTML: 

	
	    // Cria duas variáveis ​​globais    
	  
	    var mensagem = "mensagem global fora";    
	  
	    var otherGlobalVariable = "outra variável global";    
		  
	    // Criar uma função com uma variável local    
	  
	    // Ter o mesmo nome que a primeira variável global,    
	  
	    função getMessage ()    
	  
	    {    
	  
	    var mensagem = "dentro variável local";    
	  
	    // A função de alerta a variável mensagem    
	  
	    alert (mensagem);    
	  
	    // E também alerta o segunda variável global    
	  
	    alert (otherGlobalVariable);    
	  
	    // A função termina aqui    
	  
	    }    
	  
	    // Chamar a função    
	  
	    getMessage ();    
		  
	    // Alerta a variável mensagem    
	  
	    // (Que uma será, local ou global?)    
	  
	    alert (mensagem);

Salve seu trabalho e visualize a página em seu navegador. Como você pode ver, a função alert() não tem acesso à variável mensagem dentro da função . Por outro lado, a função pode ter acesso tanto ao seu variável local e para qualquer variável no espaço global.

Isso pode parecer um pouco confuso no início, mas é tudo uma questão de praticar suas habilidades de codificação tão frequentemente como você pode. A coisa importante a lembrar é que, se, por vezes, as variáveis ​​parecem não ter os valores esperados, pode ser um bug relacionado ao escopo das variáveis.
