Lição 7: Comandos de Repetição

Seus scripts JavaScript podem fazer coisas inteligentes como calcular descontos, comparação de valores, e fazer escolhas. No entanto, não se esqueça que estamos lidando com linguagem de máquina, e as máquinas são muito boas em fazer algumas coisas que nós seres humanos não somos. Computadores são bons em fazer coisas repetitivas. Você pode achar que é difícil de acreditar, mas o JavaScript ama repetição. Tanto que ele tem vários comandos de repetição . Aqui está o que você vai aprender nesta lição:

  • for ;
  • while ;
  • do ... while .

Nesta lição, você também vai aprender a:

  • usar getElementById (Id elemento) para manipular um elemento HTML com JavaScript;
  • gerar números aleatórios com random() e floor(número) ;
  • inserir HTML mark-up dinamicamente com innerHTML .

Loops

Às vezes você terá que repetir algum pedaço de código um número de vezes:

por exemplo : uma página web tem 10 botões de rádio e seu script precisa descobrir qual botão foi marcado pelo usuário.

Para isso, o script tem que passar por cima de cada botão de opção e verificar se o seu atributo checked é definido como verdadeiro ou falso. Isso significa ter que escrever este tipo de código de verificação 10 vezes?

Felizmente, isso não acontece. Você pode colocar o código em um loop, e ele vai executar qualquer número de vezes que você definir para ele.

Qualquer circuito é feito de quatro partes básicas :

  1. o valor inicial

    Um valor inicial é atribuído a uma variável, geralmente chamado de i (mas você pode chamá-lo de qualquer coisa que você quiser). Esta variável atua como contador para o loop.

  2. o valor final ou condição de teste

    O circuito precisa de um limite a ser definido: ou um número definido (Loop 5 vezes) ou uma condição de verdade (loop até que essa condição é avaliada como verdadeira). Na falta deste, você corre o risco de desencadear uma loop infinito . Isso é muito ruim: é uma repetição interminável do mesmo código, que pára navegadores dos usuários de responder. Evitar loops infinitos a todo custo, certificando-se de definir uma condição de parada para seus loops;

  3. a ação ou código a ser executado

    Você digita um bloco de código uma vez e ele vai ser executado o número de vezes entre o valor inicial eo valor final ;

  4. o incremento

    Esta é a parte que move o ciclo para a frente: o contador de inicializar tem que se mover para cima (ou para baixo no caso de você optar por looping para trás). Enquanto o circuito não atinge o valor final ou a condição de teste não for satisfeita , o contador é incrementado (ou decrementado). Isso geralmente é feito usando operadores matemáticos.

for

Este tipo de circuito, bem ... percorre um bloco de código um determinado número de vezes. Escolha um loop for , se você sabe de antemão quantas vezes o script deve ser executado.

Aqui está a sua estrutura básica: 

	
	    // Loop para o número de vezes entre o valor inicial e o valor final    
	  
	    // Incrementar o valor a cada vez que o laço for executado   
	  
	    // Até que o limite seja atingido    
	  
	    for (var = startValue; var <= endvalue; var = var + incremento)  
	  
          {    
	  
               // Código a ser executado    
	  
          }

while

Se você não sabe o número exato de vezes que o código é suposto executar, deve usar um loop while .

Com um loop while o código é executado enquanto uma condição é verdadeira , logo que esta condição for avaliada como false, o loop while paradas.

Aqui está a sua estrutura básica: 

	
	    // Loop enquanto valor inicial é inferior ou igual ao valor final    
	  
	    //     Tome nota: se a condição for avaliada como falsa desde o início,    
	  
	    //     o código nunca será executado    
	  
	    while (variável <= endvalue)    
	  
          {    
	  
                      // Código a ser executado    
	  
          }

do ... while

Este tipo de circuito é semelhante ao circuito while . A diferença entre os dois é a seguinte:

No caso do while , se a condição de teste é falso, desde o início , o código no loop nunca será executado .

No caso do do ... while , a condição de teste é avaliada após o loop ter realizado o primeiro ciclo. Portanto, mesmo se a condição de teste é falso, o código no loop executará uma vez .

Aqui é a estrutura básica de um do ... while: 

	
	    // O comando é dado     antes a condição de teste é verificado    
	  
	    do    
	  
	    {    
	  
	    // Código a ser executado    
	  
	    }    
	  
	    //     condição é avaliada neste momento     :    
	  
	    // Se é falso o ciclo para aqui    
	  
	    while (variável <= endvalue)

Experimente: adicionar número aleatório de imagens com um loop

Tempo para se codificação: preparar um documento fresco HTML com uma tag div e um id de "wrapper" e adicionar a magia JavaScript a seguir:

	  <html>

	  <div id="wrapper"> </ div>    
	      
	  <script type="text/javascript">    
	  
	    // Pegue o div html por seu atributo id    
	  
	    // E armazená-lo em um recipiente chamado var    
	  
	    var recipiente = document.getElementById ("wrapper");    
	  
	    // Usar o Math.random para gerar um número aleatório    
	  
	    // Entre 0 e 1.   Armazenar número aleatório em um var    
	  
	    var randomNumber = Math.random ();    
	  
	    // Uso Math.floor () para arredondar o número aleatório    
	  
	    // Para um valor inteiro entre 0 e 10 e armazená-lo em uma var    
	  
	    numImages var = Math.floor (randomNumber * 11);    
	  
	    // Apenas para fins de teste, permite alertar o número resultante aleatória    
	  
	    alert (numImages);    
	  
	    // O loop será executado para o número de vezes indicado    
	  
	    // Pelo número aleatório resultante    
	  
	    for (var i = 1; i <= numImages; i + +)    
	  
	    {    
	  
	    // Código a ser executado:    
	  
	    // Cria uma var e armazenar uma sequencia feita de um novo elemento img HTML    
	  
	    // E a variável de iteração i (usar + operador de concatenação):    
	  
	    // I contém o número de iteração    
	  
	    //     Tome nota: certifique-se o img src corresponde a um arquivo de imagem real    
	  
	    var newimage = '<img src="loop.gif" alt="#" />' + i;    
	  
	    // Var usar o recipiente que armazena o elemento div    
	  
	    // E usar     innerHTML     para inserir a cadeia armazenada na var newimage    
	  
	    / Cessão de atalho / Usar + = para valores anteriores não podem ser apagadas    
	  
	    container.innerHTML + = newimage;    
	  
	    }    
	    
	  </ Script>    
		    
	  </ Body>      
	  </ Html>

Perguntas, perguntas, perguntas

Eu sei, não há muito a perguntar sobre o código de exemplo acima. Mais especificamente: 1) O que é isso document.getElementById (Id elemento) ? 2) Quais são Math.random () e Math.floor (número) ? E, finalmente, 3) o que é innerHTML ? Vamos abordar cada questão, por sua vez:

  1. Se apossar de elementos HTML com document.getElementById (Id elemento)

    JavaScript pode executar todos os tipos de magia em qualquer elemento HTML uma vez que se apossar dela. Isto pode ser feito em mais de uma maneira. No entanto, a maneira mais simples é usando o document.getElementById (Id elemento) de comando. O interpretador JavaScript lê como segue:

    "Ei navegador, encontrar o elemento HTML com o nome ID como a entre parênteses".

    Porque o JavaScript geralmente faz algo com o elemento, este fica convenientemente armazenados dentro de uma variável. O que você fez no código de exemplo acima foi de agarrar a div com o atributo id chamado invólucro e armazená-lo em uma variável chamada recipiente .

  2. Gerar números aleatórios com Math.random () e Math.floor (número)

    JavaScript foi construído com comandos para executar operações matemáticas complexas de forma rápida e facilmente.

    Math.random () gera um número aleatório entre 0 e 1. Experimente-o sobre o seu próprio digitando var randomNum = Math.random (); alerta (randomNum); dentro de uma tag <script>.

    Math.floor (número) Arredonda um número para baixo para o número inteiro mais próximo e retorna o resultado . O interpretador JavaScript lê como segue:

    "Ei navegador, ter o número entre parênteses e volta-la para o próximo número inteiro."

    Experimente-o sobre o seu próprio digitando var roundedNumber = Math.floor (6.2), alerta (roundedNumber); dentro de uma tag <script>. Ficar mudando o número entre parênteses e ver como JavaScript retorna o número inteiro correspondente.

    No exemplo anterior, é usado Math.floor () juntamente com Math.random (número) para gerar um número aleatório inteiro entre 0 e 10 (digitada 11 em vez de 10, porque o número redondo retornado por Math.random () inicia a 0 não 1).

  3. Adicionar nova mark-up dentro de elementos HTML com innerHTML

    Um script JavaScript pode fazer elementos HTML aparecer e desaparecer, mudar de lugar ou de cor, e muito mais.

    Há mais de uma maneira de adicionar novos mark-up em seu documento HTML, mas usando innerHTML torna muito fácil e simples. O interpretador JavaScript lê o comando da seguinte forma:

    "Ei navegador, pegue a corda para o direito do operador de atribuição e inseri-lo no código HTML mark-up. "

    O que você fez no código de exemplo acima foi tomar a variável que armazena o elemento div wrapper e aplicando innerHTML a ele para inserir uma tag img dentro do div.

Experimente: adicionar imagens para a página com um loop while

Utilizar o mesmo documento HTML a partir do exemplo anterior, mas excluir todo o JavaScript entre o <script> ... </ Script>. Agora, digite o seguinte código JavaScript: 

	    
	  <script type="text/javascript">    
	  
	    // Pegue o div html por seu atributo id    
	  
	    // E armazená-lo em um recipiente chamado var    
	  
	    var recipiente = document.getElementById ("wrapper");    
	  
	    // Var criar o que contém o contador    
	  
	    // E dar-lhe um valor inicial de 0    
	  
	    numImages var = 0;    
	  
	    // Iniciar o loop: enquanto que o número de imagens é inferior ou    
	  
	    // Igual a 10 (ciclo começa em 0 e não 1, portanto digite 9 não 10),    
	  
	    // Ciclagem e aumentar o contador em 1    
	  
	    while (numImages <= 9)    
	  
	    {    
	  
	    // Este é o bloco de código a ser executado:    
	  
	    // Construir a cadeia para inserir no documento HTML    
	  
	    // E armazená-lo no var newImage    
	  
	    //     Tome nota: certifique-se o img src corresponde a um arquivo de imagem real    
	  
	    var newimage = '<img src="loop.gif" alt="#" />' + numImages;    
		  
	    // Var usar o recipiente que armazena o elemento div    
	  
	    // E usar     innerHTML     para inserir a cadeia armazenada na var newimage    
	  
	    / Cessão de atalho / Usar + = para valores anteriores não podem ser apagadas    
	  
	    container.innerHTML + = newimage;    
	  
	    // Incrementa o contador em 1 usando o operador de incremento    
	  
	    numImages + +;    
	  
	    }    
	    
	  </ Script>

Experimente: adicionar imagens para a página com um do ... while

Utilizar o mesmo documento HTML como no exemplo anterior. Além disso, o código JavaScript é muito semelhante ao exemplo anterior: basta mover o while (condição) fora do bloco de ação e tipo se em seu lugar, como no exemplo abaixo: 

	    
	  <script type="text/javascript">    
	  
	    var recipiente = document.getElementById ("wrapper");    
	  
	    numImages var = 0;    
	  
	    do   
	  
	    {    
	  
	    // Este bloco é executado em primeiro lugar:    
	  
	    var newimage = '<img src="loop.gif" alt="#" />' + newimage;    
	  
	    container.innerHTML + = newimage;    
	  
	    numImages + +;    
	  
	    }    
	  
	    // Aqui é a condição para avaliar:    
	  
	    // O     primeiro ciclo já foi executado     .    
	  
	    while (numImages <= 9);    
	    
	  </ Script>

Executar um do ... while com uma condição falsa

Faça apenas uma pequena mudança para o código acima: mais uma vez, substituir numImages var = 0 ; com numImages var = 12; . Agora, a condição inicial é falsa desde o início : o contador é um número maior do que 10.

Neste caso, a condição é falsa eo loop pára depois da primeira iteração completa .
