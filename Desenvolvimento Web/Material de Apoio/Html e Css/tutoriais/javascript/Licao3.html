Lição 3: Eventos

Por Maria Antonietta Perna

A web é um ambiente dinâmico, onde um monte de coisas acontecem . Mais apropriadamente, eles são chamados de eventos . Alguns desses eventos, como um usuário clicar em um botão ou mover o mouse sobre um link, são de grande interesse para um codificador de JavaScript.

Até o final desta lição você saberá:

  • o que são eventos;
  • o que o JavaScript pode fazer com eventos

O que são eventos?

Eventos são ocorrências que têm lugar no contexto das interações entre servidor web , navegador web , e usuário da web .

Por exemplo, se você estiver em uma página web e clicar em um link, há pelo menos 3 eventos importantes que estão sendo desencadeadas:

  1. onClick : desencadeado por você enquanto você clica no link;
  2. onUnload : desencadeado pelo navegador web, quando ele deixa a página da Web atual;
  3. onLoad : desencadeada pelo navegador como o novo conteúdo da página web é carregada.

Quais os eventos que você deveria focar?

Tudo depende do programa JavaScript que você está escrevendo, ou seja, sobre os objetivos que pretende alcançar com o seu script.

No entanto, os eventos mais comuns que você provavelmente vai tratar em sua JavaScript são:

  • onLoad onUnload / ;
  • onClick ;
  • onSubmit (acionado pelo usuário ao enviar um formulário);
  • onFocus / onBlur (acionado por um usuário como, por exemplo, ele clica em uma caixa de texto ou cliques de uma caixa de texto, respectivamente);
  • onMouseOver / onMouseOut (acionado pelo usuário ao mover o mouse sobre ou fora de um elemento HTML, respectivamente).

Neste tutorial, serão cobertos apenas os eventos listados acima.

O que eu faço com os eventos de um ponto de vista JavaScript?

Quando você escreve um programa em JavaScript, os eventos se tornam interessantes, porque dão ao seu script um gancho para ganhar o controle sobre o que acontece na página web.

Uma vez que seu script tem de segurar o gancho fornecido pelo evento, o script é o chefe. O jargão para isso é programação orientada a eventos : um evento acontece e JavaScript manipulá-o , por exemplo, exibir uma caixa de alerta com uma mensagem para o usuário.

Hey, um evento onClick acontecendo!

Conclusão: eventos curvar aos comandos JavaScript por meio de manipuladores de eventos . Estas são afirmações no seu script JavaScript que estão devidamente ligados a esses eventos.

Como lidar com eventos JavaScript?

O alert () é um comando que faz parte da linguagem JavaScript. O interpretador de JavaScript no navegador traduz ao longo destas linhas:

"Ei, navegador, exibir uma caixa de alerta que contém a mensagem digitada dentro do () colocando entre parênteses"

( Aviso: O JavaScript é case sensitive. Se você escreve um L em vez de um !, o script não funcionará).

Como você viu na lição anterior, simplesmente por escrever uma declaração de JavaScript entre <script> ... </ Script> a execução dessa declaração dispara o evento onLoad.

No entanto, os seus scripts JavaScript são capazes de fazer coisas mais interessantes quando se trata de eventos em resposta às ações do usuário, por exemplo um evento onClick. Como fazemos isso?

Experimente: lidar com um evento onClick com JavaScript

O navegador já tem suas próprias maneiras de manipulação de eventos . Por exemplo, quando uma página é carregada, o navegador dispara o evento onLoad e exibe o conteúdo da página, quando um usuário clica em um link, o navegador se comunica com o servidor para acessar a página solicitada, etc Estas são chamados de ações padrão .

A diversão de estar no comando, porém, é não deixar o navegador fazer o que gosta , mas deixar que o JavaScript faça o seu trabalho e decida o que é para ser feito .

A maneira mais simples para anexar um manipulador de eventos para um evento é inserir o código JavaScript necessário dentro do elemento HTML que produz o evento. Vamos ter ir simplesmente impedindo a ação padrão do navegador quando o usuário clica em um link na página. Dispare seu editor de texto e vamos começar a codificação!

Prepare um documento básico HTML para exibir um link simples, como o mostrado abaixo: 

	     
	  <! DOCTYPE html>    
	  <html>    
	  <head>    
	  <title> Lição 3: Eventos e manipuladores de eventos </ title>    
	  </ Head>    
	  <body>    
	  <h1> Lição 3: Eventos e manipuladores de eventos </ h1>   
		   
	  href="http://html.net"> <a Clique em Mim! </ a>   
		   
	  </ Body>    
	  </ Html>

Execute a página no navegador. Se você clicar no link agora, você vai direto para o site HTML.net. Esta é a ação padrão do navegador, como explicado acima.

Agora volte para o documento HTML e digite o comando JavaScript a seguir dentro do <a> tag, como segue: 

	     
	  <! DOCTYPE html>    
	  <html>    
	  <head>    
	  <title> Lição 3: Eventos e manipuladores de eventos </ title>    
	  </ Head>    
	  <body>    
	  <h1> Lição 3: Eventos e manipuladores de eventos </ h1>   
		   
	  <A href = "http://html.net"   onclick = "alert ('Não vai a lugar nenhum tão rápido?'); return false;"   >! Clique em Mim </ a>   
		   
	  </ Body>    
	  </ Html>

Vá em frente, execute a página no navegador, e ... você está preso! JavaScript está no controle!

O que aconteceu lá?

Isso mostra como é fácil assumir o controle de um link com JavaScript! Tudo o que você fez foi inserir um par de instruções JavaScript à onclick atributo da marca <a> HTML.

Você já sabe tudo sobre o alert(), então eu vou pular esta parte. O return false; diz ao interpretador de JavaScript para retornar um valor, neste caso, o valor é igual a falso, o que impede o navegador de executar a ação padrão. Você estará usando este comando pouco, muitas vezes em sua vida de programação JavaScript.

Você lida com os outros eventos listados acima, da mesma forma: basta inserir o comando JavaScript como o valor do onfocus, onblur, onmouseover, onmouseout e atributos onsubmit de um elemento HTML.

Resumo

Nesta lição você aprendeu o que são eventos, porque os eventos são importantes na programação JavaScript, e como usar manipuladores de eventos para permitir que o JavaScript esteja no controle do comportamento da sua página web.

Mas esta é apenas uma amostra minúscula do que o JavaScript pode fazer uma vez que está no comando. Siga em frente para descobrir mais.
